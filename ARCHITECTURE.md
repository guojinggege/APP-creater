## 数据流

### 用户认证流程

```
用户输入凭证 → Zustand Auth Store → API 请求
    ↓
 后端验证
    ↓
返回 Token → 本地存储 → 更新 Auth State
    ↓
导航到主应用
```

### 内容加载流程

```
用户打开 Feed → React Query 查询
    ↓
Content Store 检查缓存
    ↓
缓存命中 → 返回数据 | 缓存未命中 → 发起 API 请求
    ↓
后端返回内容
    ↓
Zustand 更新 Content Store
    ↓
组件重新渲染，显示内容
```

## 项目结构详解

### `/src/app`
Expo Router 使用文件系统路由，自动生成导航结构。

```
app/
├── _layout.tsx           # 根布局，处理认证状态
├── auth/                 # 认证路由组
│   ├── _layout.tsx       # 认证堆栈布局
│   ├── index.tsx         # 登录页面
│   └── forgot-password.tsx
└── (tabs)/              # 标签页面组
    ├── _layout.tsx      # 底部导航
    ├── index.tsx        # 首页
    ├── discover.tsx     # 发现
    ├── chat.tsx         # 聊天
    ├── creator.tsx      # 创作者
    └── profile.tsx      # 个人资料
```

### `/src/components`
可复用的 UI 组件库。

```
components/
├── Button.tsx           # 按钮组件
├── Card.tsx             # 卡片组件
├── Avatar.tsx           # 头像组件
├── Loader.tsx           # 加载器
└── index.ts             # 导出
```

### `/src/store`
Zustand 状态管理店铺。

```
store/
├── authStore.ts         # 认证状态
├── contentStore.ts      # 内容状态
└── notificationStore.ts # 通知状态
```

### `/src/utils`
工具函数和辅助文件。

```
utils/
├── format.ts            # 格式化函数
└── icons.tsx            # 图标组件
```

### `/src/types`
TypeScript 类型定义。

```
types/
└── index.ts             # 所有类型定义
```

## 技术栈选型理由

### React Native + Expo
- ✅ 一套代码支持 iOS、Android、Web
- ✅ 热重载和快速开发
- ✅ 丰富的原生模块生态
- ✅ 零配置初始设置

### Expo Router
- ✅ 基于文件系统的直观路由
- ✅ 类似 Next.js 的开发体验
- ✅ 自动深度链接支持
- ✅ 原生平台集成

### Zustand
- ✅ 极简的 API
- ✅ 无样板代码
- ✅ 小体积（< 1KB）
- ✅ TypeScript 友好

### NativeWind
- ✅ 在 React Native 中使用 Tailwind CSS
- ✅ 一致的样式系统
- ✅ 响应式设计支持
- ✅ 暗色主题内置

### React Query
- ✅ 强大的服务器状态管理
- ✅ 自动缓存和重新验证
- ✅ 乐观更新支持
- ✅ 开箱即用的错误处理

## 性能优化

### 1. 代码分割
- Expo Router 自动进行路由级别的代码分割
- 按需加载页面代码

### 2. 图片优化
- 使用 `Image` 组件的 `source` 预加载
- 实现虚拟化列表进行长列表优化

### 3. 状态优化
- Zustand 自动进行选择器优化
- 避免不必要的重新渲染

### 4. 网络优化
- React Query 的缓存机制
- 请求去重

### 5. 打包优化
- Tree-shaking 移除未使用代码
- 最小化文件大小
